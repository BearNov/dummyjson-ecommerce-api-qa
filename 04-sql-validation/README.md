# SQL Validation

This folder will contain SQL-based validation logic for API-like e-commerce data.

DummyJSON does not provide direct access to its internal database, so this section will use a small local dataset based on realistic API response structures.

The purpose of this section is to demonstrate backend validation thinking, business-rule validation, and basic data quality checks.

## Planned Content

- Local API-like product data
- Local API-like cart data
- Local API-like user data
- Cart total validation
- Product quantity validation
- Duplicate ID checks
- Missing reference checks
- Invalid product ID checks
- Negative quantity checks
- Price and discount calculation checks
- Basic product/category analysis
